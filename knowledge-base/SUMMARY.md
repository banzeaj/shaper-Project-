# Each Partner’s Contributions

## The project was completed collaboratively by two partners, with responsibilities divided to ensure efficiency and balanced workload.

Partner 1:
Created the initial repository on GitHub
Set up the project structure using Markdown files
Wrote core content for key sections of the knowledge base
Reviewed and merged pull requests
Partner 2:
Cloned the repository and contributed additional content
Improved formatting and consistency across Markdown files
Identified and fixed errors in documentation
Assisted with version control and branch management

Both partners actively participated in editing, reviewing, and refining the final project to ensure quality and completeness.

# How Git and GitHub Were Used Throughout

Git and GitHub were essential tools for collaboration and version control during the project.

A shared repository was created on GitHub to store all project files
Each partner cloned the repository to their local machine
Work was done on separate branches to avoid conflicts
Changes were committed regularly with clear messages
Updates were pushed to GitHub and reviewed through pull requests
Pull requests were used to discuss and approve changes before merging
The main branch was kept stable and up to date

This workflow ensured smooth collaboration and prevented overwriting each other’s work.

# Challenges Faced and Lessons Learned
## Challenges

Merge conflicts: Occurred when both partners edited the same file simultaneously
Understanding Git commands: Initially confusing commands like commit, push, pull, and merge
Coordination issues: Managing who works on which file at what time
Version control mistakes: Occasionally committing to the wrong branch

## Lessons Learned
Always pull the latest changes before starting work
Use meaningful commit messages for better tracking
Communicate clearly to avoid working on the same files
Use branches properly to isolate changes
Resolve merge conflicts carefully to avoid losing work
GitHub pull requests improve code quality through review
